Linux系列文章03 - 目录和文件操作

服务器 Linux Linux文件 Linux目录 2020/08/28

创建文件和目录

touch命令：创建一个空白的文件

​ linux没有一个专门的命令 用来创建空白文件

​ touch命令的设计初衷 是 修改文件的时间戳(创建、修改时间)

​ 如果touch命令 后面跟着的 文件名是不存在的，就会创建一个文件

​ 例：touch new_file

​ 例：创建多个文件

​ touch new_file new_file_2 new_file_3

mkdir命令：创建一个目录

​ mkdir是mk和dir的缩写，mk：make(创建),dir：directory(目录)

​ 参数-p：递归创建 目录结构

​ 例：mkdir -p one/two/three

拷贝文件和移动文件

cp命令：拷贝文件或目录

拷贝文件

​ 例：拷贝 new_file 文件

​ cp new_file new_file_copy

​ 例：拷贝 new_file 文件到 其他目录

​ cp new_file one/

​ 例：拷贝 new_file 文件到 其他目录，同时重命名为 new_file_copy

​ cp new_file one/new_file_copy

拷贝目录

​ 拷贝目录，要在cp后添加 -r或-R参数，俩参数作用一样，都是递归的意思

​ 例：拷贝目录 one

​ cp -r one one_copy

使用通配符*拷贝

​ 例：cp *.txt folder : 将当前目录下 所以txt文件 拷贝到 folder目录下

​ 例：cp ha* folder ：将当前目录下 所以已ha开头的文件都 拷贝到 folder目录下

mv命令：移动文件(或目录)、重命名文件(或目录)

移动文件

​ 例：将 new_file文件 移动到 one目录下

​ mv new_file one/

移动目录

​ mv命令移动目录 不需要其他参数，和移动文件一样

​ 例：将当前目录下的 new_folder目录(包括子目录和文件)移动到one这个目录

​ mv new_folder one/

重名名文件

​ linux中，没有专门的命令用于重命名文件

​ mv命令可以重命名文件，是基于其移动文件的机制

​ 经过 mv命令 移动后，原始文件变成了新名字的文件，文件内容不变

​ 例：将new_file 重命名为renamed_file

​ mv new_file renamed_file

删除文件和目录

rm命令：删除文件或目录

rm是英语remove的意思，表示“移除”

例：删除当前目录下的 new_file_copy 文件

​ rm new_file_copy

删除多个文件 用空格隔开

常用参数

  • -i参数：向用户确认是否删除

    参数有两种回答：”y”表示确认，”n”表示取消

  • -f参数：强制删除，慎用！

  • -r参数：递归删除，可以删除目录，并递归删除 子目录和文件

    r是 recursive的缩写，表示递归

rm加上 -rf命令 极为危险

删除整个系统

​ sudo rm -rf /

​

rmdir命令：删除空目录

创建链接(快捷方式)

ln命令：创建链接

​ ln是link的缩写，用于在文件之间创建链接

Linux的文件储存：

​ 每个文件分为三部分：文件名、文件内容、权限

​ 每个文件内容被分配 一个标识号码：inode

​ 每个文件名 通过inode 绑定到文件内容

链接分类

硬链接(物理链接)：使链接内容共享同样的内容，inode相同

​ 一旦文件1和文件2之间有硬链接，修改文件 其实 修改的同一个文件

​ 硬链接缺陷：只能创建指向文件的硬链接，不能指向目录(一般情况下)

软链接(符号链接)：

​ 软链接可以指向文件和目录，对于目录一般都是用软链接

创建硬链接

​ 例：创建文件file2，使之成为 文件file1 的硬链接：ln file1 file2

​ 此时 file1 和 file2 通过相同的 inode 指向 相同的文件内容

​ 删除 file1 与或file2 对另一个文件没有影响

​ 删除任意文件，对共同指向的文件内容没有影响，不会从硬盘删除

显示文件inode

​ 命令：ls -i

创建软链接

​ 软链接才真正类似 window下的快捷方式，原理很相似

​ 例：创建file1文件的软链接file2：ln -s file1 file2

​ 此时 file1和file2 俩文件的inode号 不相同

​ file2是file1的软链接(快捷方式)，所以显示的是file1的内容

​ 如果file1被删除，file2会变成“死链接”，因为指向的文件不见了
