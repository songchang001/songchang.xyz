Linux 目录和文件操作

Linux 服务器 Linux权限 Linux用户 2020/08/30

linux系统下的用户组织

  1. root用户是会默认创建的，是系统大管家，拥有所有权限，可以安装软件、修改系统设置
  2. 日常使用，一般使用 有限权限用户，如：songchang用户
  3. 我们只在必要时才会切换root身份
  4. 这一保护措施 大大提高了 linux系统的安全性，有效防止误操作和病毒的攻击
  5. 部分命令必须切换root身份才能运行

切换root用户身份

以root身份运行命令：sudo命令

如果要运行 只有root才可以运行的命令，可以在此命令前面加上sudo命令

例：sudo date

终端会提示输入密码，此密码是当前用户的密码

切换root用户方式1：sudo su命令

​ 需要输入当前用户 密码

切换root用户方式2：su 命令

​ 需要切换root用户密码

​ 建议使用：su - 命令，切换root用户并直接跳转到root用户家目录(/root)下

切换root用户方式3：sudo -i 命令

​ 切换root用户，并直接跳转到root用户家目录下

退出root用户身份

  1. Ctrl+d
  2. exit+回车

用户管理

useradd命令：添加新用户

​ 例：useradd lch

passwd命令：修改用户的密码

​ 例：password lch

​ 然后 两次输入 要设置或修改的密码

userdel命令：删除用户

例：userdel lch

​ 当前命令会删除用户，但是不会删除用户的家目录 /home/lch

删除用户并删除用户家目录：

​ userdel -r lch 或 userdel –remove lch

用户群组

linux中每个用户都属于一个特定的群组

如果不设置用户群组，默认会创建一个和它用户名一样的群组，并把用户划归到这个群组

创建群组

​ 例：groupadd friends

修改用户账户：usermod命令

​ usermod是 user modify的缩写，modify是修改的意思，用户修改用户账户

​ 参数：

  1. -l参数：重命名

    重命名后，/home中的家目录名保持不变，需要手动修改

  2. -g参数：修改用户所在群组

    例：usermod -g friends songchang

  3. -G参数：一次性将用户 添加多个群组

    例：usermod -G friends,happy,funny lch

    ​ 将 用户lch 添加到 friends、happy 和 funny 三个群组，群组名之间用逗号分隔，且没有空格

  4. -a参数：追加群组

    -g和-G参数 会先将用户从原先群组中移除，再追加到新的群组，如果不想移除出来，则加-a参数

    -a参数只能和 -G参数一起使用

    例：usermod -ag good lch ：将用户 lch 追加到good群组中

获知用户所属群组：groups命令

​ 例：groups lch （用户必须存在）

​ groups不加任何参数执行，会显示 当前用户所在群组

删除群组：groupdel命令

​ 例：groupdel good ：删除 good群组

改变文件所有者：chown命令

​ chown是change owner的缩写，需要root身份才能运行

​ 例：chown lch file.txt ：将file.txt的所有者改为lch用户

​ 执行后，所有者变为lch用户，群组不变

​ 例：chown lch:friends file.txt ：将file.txt文件的所有者改为lch，群组改为friends

​ 参数：

​ -R参数：递归改变 文件及子文件的 所有者和群组

​ 例：chown -R songchang:songchang /home/lch

​ 将lch用户的 家目录和所有子目录、子文件都 占据，让lch用户 无家可归

改变文件群组：chgrp命令

​ chgrp是change group的缩写，用于改变文件群组，需要root身份才能运行

​ 例：chgrp lch file.txt ：将file.txt文件的群组改为lch

访问权限

​ linux系统中，每个文件和目录都有一列权限属性

​ 访问权限指明了谁有读的权限，谁有修改的权限，谁有运行的权限

​ 在运行命令“ls -l”时，返回结果的第一列就是访问权限

​

第一列包含 d、r、l、x、w等字母，可以统称为文件访问权限符

文件访问权限符含义

d：directory的缩写，表示目录

-：普通文件

l：link的缩写，表示链接

r：read的缩写，表示可读

w：write的缩写，表示 可写/修改

x：execute的缩写，表示 可执行/运行

​ 如果x权限在目录上，表示这个目录可读，可查看其子目录

如果对应位置有字母，表示有对应权限；对应位置为“-”，表示没有对应权限

第11位上的”.”：https://blog.csdn.net/dashuai03091199/article/details/38920833

权限符 划分

第一组rwx：文件所有者对此文件的访问权限

第二组rwx：文件所属群组其他用户对此文件的访问权限

第三组rwx：初前两组的其他用户对此文件的访问权限

如果有w权限(写的权限)，那么也有删除此文件的权限

解析 文件file.txt 的权限含义：-rw-rw-r–

root用户是超级用户，它有linux系统所有权限，可以读、写、运行所有文件

修改文件访问权限：chmod命令

chmod命令不需要root用户才能运行，只要你是文件的所有者，就可以修改文件的访问权限

chmod是change mode的缩写，用于修改文件的各种访问权限
